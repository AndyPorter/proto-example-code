Branching dependent on checkboxes

If one or more checkboxes are checked branch to page a, if no checkboxes are checked branch to page b.

Idea/need from Ros - a list of existing people is displayed, the user selects all the people who meet the criteria. If no one is selected when the page is submitted take the user to a page where they can add a new person.

Previous designs asked the user to 1) tick another checkbox if none of the existing people met the criteria, and 2) a link to add another person to the list. Testing the new idea with users seems to work better, so we now need to make something more realistic - that can be used without intervention

Challenges were:

  • the list of people is built dynamically - so we don't know how many checkboxes will be on the page - between 1 and 4.
  • we'd like code anyone could easily re-use in their proto - code easy to read and reuse
  • we don't have a working example to copy/adapt - the kit has a branching example for a radio group
