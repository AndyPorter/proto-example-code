Branching dependent on checkboxes

If one or more checkboxes are checked branch to page a, if no checkboxes are checked branch to page b.

Idea/need from Ros - a list of existing people is displayed, the user selects all the people who meet the cirteria. If no one is selected when the page is sumbitted take the user to a page where they can add a new person.

Previous designs asked the user to 1) tick another checkbox if none of the existing people met the criteria, and 2) a link to add another person to the list. Testing the new idea with users seems to work better, so we now need to make something more realistic - that can be used without intervention

Challenges for us are:

  • the list of people is built dynamically - so we don't know how many checkboxes will be on the page - between 1 and 4
  • we don't have a working example to copy/adapt - the kit has a branching example for a radio group

We don't know if its better to do the work all server side (in the routes file) - our preference, for consistency with other logic. Or to do some work client side - perhaps a script that runs onsubmit of the page setting a hidden variable if any checkboxes are checked - which can then be read server side to route to page a or b. For a protoype it doesnt matter as long as it works - but its better if its consistent and easy for others to follow
